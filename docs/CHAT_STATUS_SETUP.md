# Настройка статусов в чате (в сети / недавно / печатает)

Чтобы статусы в чате работали на сайте, сделайте следующее.

## 1. Правила Firestore

В [Firebase Console](https://console.firebase.google.com) откройте проект → **Firestore Database** → вкладка **Rules**.

Убедитесь, что разрешены:

- **Чтение документов `users/{userId}`** — для отображения статуса «В сети» / «был(а) недавно» у собеседника.
- **Запись в свой документ `users/{userId}`** — приложение обновляет поле `last_seen` у текущего пользователя.
- **Чтение и запись коллекции `typing`** — для индикатора «печатает».

Пример фрагмента правил (добавьте к уже существующим или объедините логику):

```
// Пользователи: читать может любой авторизованный, писать — только свой документ
match /users/{userId} {
  allow read: if request.auth != null;
  allow create, update, delete: if request.auth != null && request.auth.uid == userId;
}

// Индикатор набора текста
match /typing/{conversationId} {
  allow read, write: if request.auth != null;
}
```

В корне проекта есть файл **`firestore.rules`** — его можно развернуть через Firebase CLI:

```bash
firebase deploy --only firestore:rules
```

(Предварительно: `firebase init` и выбор проекта, если ещё не настроено.)

## 2. Деплой фронтенда

Чтобы изменения кода попали на хостинг (например, Vercel):

1. Закоммитьте и запушьте изменения в репозиторий.
2. Дождитесь сборки и деплоя (Vercel делает это автоматически при push).

Либо соберите и залейте вручную:

```bash
npm run build
# затем загрузите содержимое папки dist на хостинг
```

## 3. Проверка

- Откройте сайт в двух браузерах (или в обычном и в режиме инкогнито) под разными пользователями.
- Откройте чат и выберите друг друга в личном диалоге.
- Под именем собеседника должно отображаться «В сети» или «был(а) недавно».
- При наборе сообщения у одного пользователя у второго под именем должно появляться «• • • печатает».

Если статусы не обновляются, откройте консоль браузера (F12) и проверьте ошибки; чаще всего это блокировка из‑за правил Firestore.
